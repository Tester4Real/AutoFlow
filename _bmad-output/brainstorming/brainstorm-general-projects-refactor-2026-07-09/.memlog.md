- (direction by user) Refactor AutoFlow from one-character/one-channel workflow into general project-based workflow with optional characters, places, uploaded reference assets, editable project asset library, JSON-selected prompt references, videos added to queue rather than auto-generated, and image review page to choose best variants.
- (decision by user) Use default facilitator-chosen brainstorming batch: Morphological Analysis, Decision Tree Mapping, Failure Analysis, Disney Method.
- (technique by coach) Started Morphological Analysis: split general-project refactor into independent design dimensions and option sets.
- (idea by coach) Use Project as the top-level container with name, slug/folder, Flow project/account state, default settings, asset library, batch history, and review state.
- (idea by coach) Use typed reusable assets: characters, places, props, styles, generic references, and possibly reference frames, with projects allowed to have zero characters.
- (idea by coach) Make JSON prompt records refer to project assets by stable IDs/names rather than hardcoded Jack reference, allowing per-prompt reference inclusion.
- (idea by coach) Separate image generation from video generation: image prompts generate variants; video prompts are planned and added to queue, not auto-run.
- (idea by coach) Add review/selection workspace where generated image variants can be chosen as final, updating canonical filename/reference mapping without losing alternates.
- (decision by user) Image review and selection should live in a larger full extension page/tab, not only inside the narrow Chrome side panel.
- (technique by coach) Started Decision Tree Mapping: trace project creation/editing, JSON reference resolution, image variant selection, and video queue readiness.
- (idea by coach) Use side panel as lightweight operations cockpit and larger extension page as Project Studio for asset management, image review, and video queue preparation.
- (idea by coach) Project Studio sections could be Project Settings, Assets, Import/Map JSON, Image Review, Video Queue, Gallery/Downloads.
- (idea by coach) A JSON import can either resolve references to project assets, create unresolved reference warnings, or create placeholder assets for later upload.
- (idea by coach) Treat video jobs as draft/ready queue items created from selected image variants and animation prompts, never auto-running immediately after image generation.
- (idea by coach) Default video reference strategy should likely use selected image as start frame only; extra character/place refs can be opt-in per project/job if Flow supports it reliably.
- (decision by user) Missing JSON-referenced assets should not auto-create placeholder uploads; user should manually upload/resolve the asset.
- (technique by coach) Started Failure Analysis: identify how project asset refs, JSON import, image selection, video queue, and Flow upload caches could fail.
- (risk by coach) If missing JSON references are only warnings, users may accidentally generate prompts without required character/place references.
- (idea by coach) Import should allow unresolved references but mark affected prompts/jobs as blocked until required assets are manually uploaded and resolved.
- (risk by coach) Renaming assets or files after batches exist can break prompt mappings, gallery selections, downloaded filenames, and video start-frame links.
- (idea by coach) Use stable asset IDs and immutable prompt IDs internally; display names and file names can change without breaking old job references.
- (risk by coach) Selected image variants can become inconsistent with final filenames if the extension renames immediately or loses the original variant mapping.
- (idea by coach) Store canonical selected-variant metadata first; perform final filename/export operations explicitly and keep alternates retained.
- (risk by coach) Flow media/upload IDs may be account/project-specific, so cached reference uploads can become invalid after switching Google account or Flow project.
- (idea by coach) Cache uploaded reference media per project plus Flow account/project context, and reupload/relink when current context changes.
- (risk by coach) Video generation may not honor the same reference-image behavior as image generation, causing continuity surprises.
- (idea by coach) Default video jobs to selected image as start frame only; expose advanced reference modes later after testing Flow behavior.
- (decision by user) JSON-listed references are required by default; if any listed asset reference is missing or unresolved, block the affected prompt until manually resolved.
- (technique by coach) Started Disney Method: Dreamer, Realist, Critic pass for project-based asset/refactor UX.
- (idea by coach) Dreamer: AutoFlow becomes Project Studio where user builds reusable project worlds with characters, places, props, styles, imports scene JSON, reviews variants, then prepares video jobs deliberately.
- (idea by coach) Dreamer: UI feels like a production board: assets are cast/locations/props, image prompts are shots, variants are takes, selected images become final frames, videos become planned shots queued for render.
- (idea by coach) Realist: Add a full extension page for Project Studio while keeping side panel as operations cockpit for current project, queue status, run/stop/retry, and open-studio entry point.
- (idea by coach) Realist: Phase implementation into data model/storage, project/asset CRUD, JSON import resolver, image review selection, video queue builder, then export/finalize downloads.
- (idea by coach) Realist: Store stable project_id, asset_id, prompt_id, variant_id, and job_id; display names and file names are mutable labels, not primary identity.
- (risk by coach) Critic: Scope can balloon if project studio tries to solve asset management, queue orchestration, gallery, exports, and Flow account cache in one release.
- (idea by coach) Critic mitigation: Define MVP as project asset library + JSON refs + blocked missing refs + image variant review + manual video queue creation; leave advanced video reference modes and packaged export polish for later.
